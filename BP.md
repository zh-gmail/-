# AI 美学工坊 — 商业计划书

---

## 一、问题定义

### 1.1 核心痛点

| 痛点 | 描述 |
|------|------|
| **形象决策成本高** | 换发型/妆容/穿搭需要实际到店或购买，试错成本高、时间长 |
| **缺乏可视化预览工具** | 现有工具多为美颜滤镜（磨皮、瘦脸），缺乏对发型/妆容/穿搭的实质性变换能力 |
| **AI 换装门槛高** | Stable Diffusion / Midjourney 等技术门槛高，需繁琐的 prompt 调优和 ControlNet 配置 |
| **身份保持不一致** | 多数 AI 换图工具会改变人物面部特征，生成结果"不像本人" |
| **素材管理分散** | 灵感收集散落在小红书、Pinterest 等平台，缺乏统一管理和分类 |

### 1.2 目标用户

- **个人消费者（C 端）**：有形象管理需求、关注时尚穿搭的年轻群体（18-35 岁）
- **美业从业者（B 端）**：理发店、造型工作室、服装电商，需要快速出效果图
- **内容创作者**：时尚博主、短视频创作者，需要大量视觉素材

## 二、市场场景分析

### 2.1 市场规模

- **全球 AI 图像生成市场**：2026 年预计达 150 亿美元，年复合增长率 35%
- **中国虚拟试穿/试妆市场**：2025 年预计超 200 亿元人民币
- **美业数字化市场**：发型/美妆/穿搭三块交叉市场，蓝海空间巨大

### 2.2 应用场景

```
 场景 A：个人形象实验
 ─────────────────────────────────────
 用户想换发型但不确定效果
 → 上传自拍 → AI 生成 5 款发型效果
 → 支持发色调整 → 保存/分享

 场景 B：美业辅助决策
 ─────────────────────────────────────
 理发师给客户看效果图
 → 现场拍照 → AI 生成多种选项
 → 客户选择后开始实际操作

 场景 C：电商搭配推荐
 ─────────────────────────────────────
 服装电商展示穿搭效果
 → 上传模特图 → AI 变换多套穿搭
 → 生成搭配推荐 → 提高转化率

 场景 D：灵感素材采集
 ─────────────────────────────────────
 看到喜欢的明星造型
 → 上传照片 → AI 提取风格特征
 → 保存到素材库 → 后续参考使用
```

### 2.3 市场趋势

1. **生成式 AI 消费化**：从专业工具向消费级应用渗透
2. **形象经济崛起**：短视频时代用户对个人形象投入持续增加
3. **隐私意识增强**：本地化/端侧 AI 处理需求上升
4. **多模态 AI 成熟**：视觉理解 + 图像生成能力大幅提升

### 3.2 六大功能模块

| 模块 | 功能 | 关键技术点 |
|------|------|-----------|
| **Ⅰ 照片换发** | 上传/拍照 → AI 分析脸型 → 生成多款发型 → 发色选择 + 参考素材 | face analysis + hairstyle edit + color presets |
| **Ⅱ 照片换妆** | 同上（无发色选择）+ 参考素材 | makeup edit prompts |

### 4.2 关键 AI 调用链路

```
用户上传照片
  │
  ▼
resizeImage() ──→ 压缩至 1024×1024, JPEG 85% 质量
  │
  ▼
callQwenVL() ──→ 【人物分析】
  │               prompt: VLM_ANALYSIS_PROMPT_CN
  │               model: qwen-vl-max
  │               output: 脸型/肤色/发型分析 + 推荐
  │
  ▼
parseFaceAnalysis() ──→ 提取发型/妆容/穿搭推荐列表
  │
  ▼
buildEditPrompts() ──→ 构造图像编辑提示词
  │                     (含 negative_prompt 防身份改变)
  │
  ▼
callQwenImageEdit() ──→ 【图像编辑】
                        model: qwen-image-2.0-pro
                        parameters: n, size, negative_prompt
                        output: 变换后的图片 URLs
```

### 4.3 身份保持机制

```javascript
negative_prompt:
  '改变性别, 改变脸型, 改变五官, 改变肤色, 改变服装, 改变背景, 变形, 扭曲, 不像本人'

prompt 模板:
  '保留此图所有特征完全不变。仅将[发型/妆容/服装]改为[目标]。[不改变其他项]。'
```

此机制确保 AI 只修改指定区域，保留人物核心身份特征。

- **L4 — 结果管理**：素材库 + 分类管理 + 对比预览

## 六、创新点

### 6.1 One-Prompt 多变换

同一张照片，AI 分析一次后，生成发型/妆容/穿搭三种变换，用户在一个页面内无缝切换对比，体验流畅。

### 6.2 结构化分析驱动生成

不是简单的"图生图"，而是先通过 VLM 做结构化分析（脸型/肤色/风格），再用分析结果指导图像编辑，生成结果更精准、更个性化。

### 6.3 三层身份保持

```
Layer 1: negative_prompt 显式约束
Layer 2: prompt 中明确要求"保留所有特征"
Layer 3: 模型本身的 identity preservation 能力（qwen-image-2.0-pro）
```

### 6.4 风格提取 + 分类管理

从照片中自动提取发型/妆容/穿搭的风格特征，结构化为可管理的素材条目，构建个人风格库。

## 九、后续迭代方向

### 9.1 近期（1-3 个月）

- **照片/参考素材实时预览** — 上传参考图时实时显示在大图中

- **生成历史** — 保存历史生成记录，支持回溯
- **分享功能** — 生成结果一键分享到社交媒体

### 9.2 中期（3-6 个月）

#### 轻量大模型前端分析

将部分 AI 分析能力从云端迁移到浏览器端，降低延迟和 API 成本：

- **人脸关键点检测** — 使用 MediaPipe / TensorFlow.js 在前端检测脸型轮廓，无需后端调用
- **肤色自动检测** — Canvas API 分析图片肤色区域，自动推荐适配发色
- **发型分割 (Hair Segmentation)** — 前端模型实时分割头发区域，辅助生成
- **边缘计算推理** — WebGL / WebGPU 加速的小模型推理

```javascript
// 前端分析流程（提案）
async function frontendAnalysis(imageData: ImageData) {
  const faceMesh = await mediapipeFaceMesh.detect(imageData); // 人脸关键点
  const skinTone = analyzeSkinTone(imageData, faceMesh);      // 肤色检测
  const hairRegion = await hairSegmentation(imageData);        // 发型分割
  return { faceMesh, skinTone, hairRegion };
}
```

优势：
- 减少 1-3 次 API 调用，降低 40% 以上成本
- 分析延迟从 3-5 秒降至 100-500ms
- 部分数据无需离开用户设备，提升隐私保护

#### 推荐系统

- 基于历史生成结果的风格偏好学习
- 脸型 ⇔ 发型匹配数据库
- "最多人选择" / "热门搭配" 社交推荐

3. **3D 试穿试戴**：3D 服装/首饰在重建模型上的真实物理效果
4. **真实光影渲染**：PBR 材质渲染，室内外不同光照效果
