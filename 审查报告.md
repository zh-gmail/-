## 无法自动修复的问题

### REPORT-1 [代码质量] `as Error` 类型断言用于 AbortError 检测

| 项 | 内容 |
|------|------|
| **位置** | `Extraction.tsx:103`、`PhotoEdit.tsx:127`、`falProvider.ts:135` |
| **类型** | 代码质量 — 不严谨的类型断言 |
| **代码** | `if ((err as Error).name === 'AbortError') return;` |
| **原因** | catch 变量默认 `unknown`，`as Error` 通过断言访问 `.name`。运行时不安全问题：若 catch 到非对象值（如 `throw "abort"`），`.name` 为 `undefined`，比较仅跳过不做 Abort 处理，不崩溃。用 `instanceof` 替代会遇到 `DOMException` 非 `Error` 子类的边界情况。 |
| **建议** | 维持现状。当前功能正确，抽取辅助函数会过度工程化。 |

### REPORT-2 [架构] `AppContext.tsx` — IndexedDB 加载失败静默降级

| **类型** | 架构 — 错误不可见 |
