## 已修复的问题

### 🔴 严重

#### #1 `openDB()` 不返回 `dbPromise` — IndexedDB 全程静默失败

| 项 | 内容 |
|------|------|
| **文件** | `src/services/libraryDB.ts:26` |
| **类型** | 运行时 Bug — 异步函数未返回值 |
| **症状** | `openDB()` 在首调用路径（`dbPromise` 为 null 时）创建 Promise 但不返回它，函数隐式返回 `undefined`。`getAllItems()` 中 `await openDB()` 得到 `undefined`，访问 `.transaction()` 抛异常，被 `.catch()` 吞掉。**素材库永远为空**，用户保存的发型全部丢失。 |
| **根因** | tsconfig 未启用 `noImplicitReturns`，TypeScript 未检查此漏返回值 |
| **修复** | 第 26 行添加 `return dbPromise;` |
| **验证** | tsc ✓ build ✓ |

### 🟡 中等

#### #2 `GestureCallback` 死类型定义

| 项 | 内容 |
|------|------|
| **文件** | `src/services/arEngine.ts:21` |
| **类型** | 代码质量 — 死代码 |
| **症状** | `GestureCallback` 类型包含 `'nod' | 'shake'`，但项目中没有任何代码能产生这两种手势。MindAR 无内置手势识别，手势识别由 `useHandTracking` 负责，其 `Gesture` 类型只有 `'up' | 'down' | 'swipe_left' | 'swipe_right' | 'palm' | 'fist'`。 |
| **修复** | 移除 `'nod' | 'shake'`，保留实际可用的 `'swipe_left' | 'swipe_right'` |
| **验证** | tsc ✓ build ✓ |

#### #3 `setCurrentHairstyleIndex` 类型不兼容回调

| 项 | 内容 |
|------|------|
| **文件** | `src/store/AppContext.tsx:13` |
| **类型** | 类型错误 — 接口定义窄于实际用法 |
| **症状** | `AppContextType` 定义 `setCurrentHairstyleIndex: (index: number) => void`，但 `LiveCamera.tsx` 多处传递回调 `(prev) => (prev + 1) % length`（React `useState` 的 Updater 模式）。TypeScript 报 `TS2345` 类型不匹配。 |
| **修复** | 类型改为 `React.Dispatch<React.SetStateAction<number>>`，同时兼容值和回调 |
| **验证** | tsc ✓ build ✓ |
